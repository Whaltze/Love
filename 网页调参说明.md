# 生日网页调参说明

这份说明用于后期自行修改字体、字号、颜色、卡片大小、动画时间、照片数量和装饰数量。

## 一、四个主要文件分别负责什么

- `index.html`：页面结构。通常只在新增一整块场景时修改。
- `style.css`：颜色、字体、字号、宽高、间距、圆角、阴影和动画外观。
- `birthday-content.js`：文字、照片、时间、动画数量与速度。日常修改优先从这里开始。
- `script.js`：交互和动画逻辑。已经分区注释，一般不需要修改。

## 二、修改字体、字号和颜色

打开 `style.css`，最上方的 `:root` 就是全站调参面板。

常用变量：

```css
--font-main: ...;                 /* 普通中文字体 */
--font-romantic: ...;             /* 情书、标题、卡片使用的浪漫字体 */
--paper: #fffbee;                 /* 爱心树背景 */
--love: #be1a25;                  /* 主红色 */
--opening-title-size: ...;        /* 开场标题字号 */
--story-font-size: ...;           /* 情书字号 */
--speech-font-size: ...;          /* 米老鼠气泡字号 */
--photo-caption-size: ...;        /* 照片上下文字字号 */
--birthday-title-size: ...;       /* 礼物卡片标题字号 */
--birthday-body-size: ...;        /* 礼物卡片正文字号 */
```

CSS 的 `clamp(最小值, 随屏幕变化值, 最大值)` 会让字号自动适配手机和电脑。例如：

```css
--birthday-title-size: clamp(25px, 4vw, 42px);
```

如果想让标题整体更小，可改为：

```css
--birthday-title-size: clamp(22px, 3.2vw, 34px);
```

## 三、让一句话尽量保持一行

不要第一时间使用 `white-space: nowrap`，因为手机上可能溢出屏幕。建议依次尝试：

1. 增大对应卡片宽度；
2. 减小字号；
3. 减小左右内边距；
4. 最后才考虑缩短句子。

照片卡片相关：

```css
--photo-card-width: clamp(228px, 25vw, 330px);
--photo-caption-size: clamp(13px, 1.28vw, 17px);
```

礼物生日卡片相关：

```css
--birthday-card-width: 530px;
--birthday-card-padding-x: clamp(32px, 5vw, 58px);
--birthday-title-size: clamp(25px, 4vw, 42px);
--birthday-body-size: clamp(14px, 1.75vw, 18px);
```

例如标题换行太多，可以先把：

```css
--birthday-card-width: 530px;
```

改成：

```css
--birthday-card-width: 610px;
```

如果只想修改手机效果，搜索 `@media (max-width: 700px)`，在手机适配区域修改对应选择器。

## 四、修改开场追逐爱心

打开 `birthday-content.js`，找到 `opening`：

```js
opening: {
  moveEveryMs: 1050,
  transitionMs: 720,
  edgePaddingPx: 24,
  exitDurationMs: 760,
  trailCount: 10,
},
```

- `moveEveryMs`：多久换一次位置。越小越难抓。
- `transitionMs`：移动过程持续多久。越大越柔和。
- `edgePaddingPx`：爱心与移动区域边缘的距离。
- `exitDurationMs`：成功点击后，开场淡出时间。
- `trailCount`：移动轨迹小爱心数量。

开场爱心尺寸在 `style.css` 修改：

```css
--opening-heart-size: clamp(82px, 10vw, 126px);
```

## 五、修改文字出现和场景停留时间

打开 `birthday-content.js`。

情书打字速度：

```js
story: {
  characterDelayMs: 75,
  invitationDelayMs: 1200,
},
```

- `characterDelayMs` 越小，打字越快。
- `invitationDelayMs` 是情书结束后米老鼠出现前的等待时间。

场景时间：

```js
timing: {
  transitionCoverMs: 1900,
  transitionRevealMs: 760,
  photoContinueButtonMs: 3600,
  photoDurationMs: 18000,
  giftOpenDelayMs: 720,
  cardRevealDelayMs: 760,
},
```

所有以 `Ms` 结尾的数字单位都是毫秒。`18000` 就是 18 秒。

## 六、添加或扩展很多张照片、动图和视频

1. 把照片、GIF 动图或视频放进 `photos` 文件夹。
2. 打开 `birthday-content.js`。
3. 在 `photos: []` 中复制一个完整照片对象。
4. 修改图片路径和上下文字。

示例：

```js
{
  src: "./photos/photo-07.jpg",
  alt: "第七张生日回忆照片",
  top: "照片上方的文字。",
  bottom: "照片下方的文字。",
},
```

可以一直追加第八张、第九张、第二十张，不需要修改 `script.js`。网页会：

- 自动生成对应照片卡片；
- 自动复制一套用于无缝循环；
- 自动编号；
- 点击后自动放大；
- 自动显示对应上下文字；
- 根据照片数量自动延长整轮流动时间。

GIF 动图与普通照片写法相同：

```js
{
  src: "./photos/memory.gif",
  alt: "一段会动的回忆",
  top: "动图上方的文字。",
  bottom: "动图下方的文字。",
},
```

MP4 / WebM 视频写法：

```js
{
  src: "./photos/memory-video.mp4",
  type: "video",
  poster: "./photos/memory-video-cover.jpg", // 可选封面
  autoplay: true,
  loop: true,
  muted: true,
  alt: "一段生日视频回忆",
  top: "视频上方的文字。",
  bottom: "视频下方的文字。",
},
```

视频类型通常会按扩展名自动识别，但明确写 `type: "video"` 更方便以后阅读。照片墙自动播放视频时应保留 `muted: true`，否则手机浏览器可能阻止自动播放；点击视频放大后，可以在播放器控件中手动取消静音。

照片流速度在 `effects` 中修改：

```js
photoSecondsPerCard: 7,
minimumPhotoTrackSeconds: 38,
photoWheelSensitivity: 1,
photoSwipeSensitivity: 1,
```

`photoSecondsPerCard` 越大，照片流越慢。
`photoWheelSensitivity` 越大，滚轮每次移动越远；`photoSwipeSensitivity` 越大，手机滑动越跟手。

照片流在没有操作时会一直按上述速度自动流动。鼠标滚轮、触控板横向滚动、鼠标左右拖动、手机左右滑动或键盘方向键都可以临时接管；操作结束后会从当前位置继续自动流动。

如果暂时没有图片，将 `src` 留为 `""`，网页会显示浪漫占位卡片。

## 七、修改转场米老鼠数量

在 `birthday-content.js` 中找到：

```js
transitionHeadRows: 8,
transitionHeadColumns: 12,
```

总数量等于行数乘列数：

- `8 × 12 = 96` 个；
- `6 × 8 = 48` 个，适合性能较弱的手机；
- `10 × 14 = 140` 个，更密集。

同一处还能修改：

```js
treeBloomCount: 700,
giftSparkleCount: 34,
finalFloatingHeartCount: 40,
```

数量越高，画面越密，但也会增加手机绘制压力。

## 八、调整礼物卡片大小与人像位置

礼物卡片在 `style.css` 顶部调整：

```css
--birthday-card-width: 530px;
--birthday-card-min-height: 430px;
--birthday-card-padding-top: clamp(102px, 12vw, 122px);
--birthday-card-padding-x: clamp(32px, 5vw, 58px);
--birthday-card-padding-bottom: clamp(32px, 5vw, 52px);
```

人像位置搜索：

```css
.card-portrait-wrap
```

常用属性：

- `top`：负数越大，人像越往上。
- `width`：人像整体宽度。
- `height`：人像容器高度。

手机端的人像尺寸在第二个 `@media (max-width: 700px)` 中单独设置。

## 九、替换图片素材

- 米老鼠透明素材：`assets/mickey-transparent-clean.png`
- 礼物卡片头饰人像：`assets/portrait-headwear-cutout.png`

如果新图片文件名不同，需要同步修改 `index.html` 中对应 `<img src="...">`。

## 十、修改后建议检查

每次调整后至少检查：

1. 电脑宽屏下是否溢出；
2. 手机宽度下标题是否过多换行；
3. 开场爱心是否仍在移动区域内；
4. 照片流是否连续；
5. 点击照片后是否暂停，关闭后是否恢复；
6. 礼物卡片和顶部人像是否超出屏幕。
